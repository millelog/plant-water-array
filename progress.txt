Plant Water Array — Feature Progress

Completed Feature Groups

1. Watering Log & Plant Notes (priority 1) — DONE
   - WateringLog model, CRUD, API routes
   - Log watering UI on PlantDetail page
   - Dashboard "last watered X days ago"
   - Auto-watering detection from moisture jumps
   - Per-sensor notes field

2. Push Notifications via ntfy (priority 2) — DONE

3. Historical Data & Insights (priority 3) — DONE

   - PlantInsights card on PlantDetail (avg moisture, drying rate, days until dry, waterings)
   - SensorReadingsGraph dual-mode: raw readings or aggregated with min/max band

4. Dashboard Quality-of-Life (priority 4) — DONE

   - Weather widget (Open-Meteo API, lat/lon from Settings, temp + humidity)
   - Kiosk mode (fullscreen overlay, auto-cycles zones, navbar toggle, Escape to exit)
   - Quick actions on PlantCard hover/tap (Log Watering, Threshold, History)

5. Mobile Experience (priority 5) — DONE

   - MobileNavContext for shared isMobile/sidebarOpen state, auto-close on route change
   - Bottom navigation bar (Dashboard, Alerts, Settings) with alert badge
   - Mobile-optimized PlantCards: larger moisture text, taller moisture bar, bigger tap targets
   - PWA manifest + service worker via vite-plugin-pwa (installable, offline API caching)
   - SVG app icons (192px, 512px), Apple mobile web app meta tags, viewport-fit=cover

6. Database Backup & Maintenance (priority 6) — DONE
   - GET /admin/backup endpoint using SQLite online backup API (safe copy while DB is in use)
   - Download Backup button on Settings page
   - Database Health section on Settings (total readings, DB size, readings/day, oldest reading)
   - GET /admin/stats endpoint with reading counts, DB file size, per-day averages
   - retention_days field on SystemConfig (default 90, configurable 7-365 days)
   - Auto-cleanup on server startup (deletes readings older than retention_days)
   - Manual cleanup button preserved for ad-hoc deletion

7. Sensor & Device Diagnostics (priority 7) — DONE
   - HeartbeatLog model (device_id, timestamp, ip_address, firmware_version)
   - Heartbeat logging hooked into existing heartbeat endpoint
   - GET /devices/{device_id} single device lookup
   - GET /devices/{device_id}/heartbeats (last 50 entries)
   - Sensor health computation (reading frequency, ADC stuck detection, flatline/variance)
   - GET /sensors/health/batch and GET /sensors/{id}/health endpoints
   - Device Detail page (/devices/:deviceId) with info card, sensors list, heartbeat history, uptime
   - "Details" button on Devices page linking to device detail
   - Health column on Sensors page (OK/Stale/ADC=0/ADC=4095/Flatline badges)
   - Debug toggle on PlantDetail: dual Y-axis chart (moisture + raw ADC), calibration reference lines
   - Raw ADC values table (last 20 readings) with calibration bounds display
   - Cascade delete updated to clean up heartbeat logs

   - admin.py DB_PATH derived from DATABASE_URL env var

   - Backend Dockerfile (python:3.12-slim, /data volume for DB + firmware)
