## Project Overview

Plant Water Array — distributed plant moisture monitoring system with ESP32 sensors, a Python backend, and a React frontend. ESP32 devices self-register via captive portal WiFi provisioning, send periodic moisture readings, and support OTA firmware updates.

## Development Commands

### Backend (FastAPI + SQLite)
```bash
cd backend
source .venv/bin/activate          # venv lives at backend/.venv
pip install -r requirements.txt
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```
API docs at http://localhost:8000/docs

### Frontend (React + Vite + TypeScript + Tailwind)
```bash
cd frontend
pnpm install
pnpm dev          # Vite dev server on port 5173
pnpm build        # TypeScript compile + Vite bundle
pnpm lint         # ESLint
```

### ESP32 Flashing
```bash
python flash_esp.py               # Auto-detect port, erase, flash, upload source
python -m mpremote connect <port> repl   # Serial console debug
```
Windows: use `python -m esptool` (not `esptool.py` directly — version mismatch issue).

### VS Code
`.vscode/launch.json` has configs: "Backend (FastAPI)", "Frontend (Vite)", "Flash ESP32", and compound "Full Stack" (backend + frontend).

## Architecture

Three independent components, no monorepo tooling:

**Backend** (`backend/`) — FastAPI, SQLAlchemy ORM, SQLite (`backend.db`, auto-created).
- Entry: `main.py` → mounts routers from `routers/` (devices, sensors, readings, alerts, firmware)
- `models.py` — ORM models: Device, Sensor, Reading, Threshold, Alert, Firmware
- `schemas.py` — Pydantic request/response models
- `crud.py` — all DB operations
- `database.py` — SQLite connection setup
- Firmware files stored in `backend/firmware_storage/{version}/`

**Frontend** (`frontend/`) — React 18, TypeScript, Tailwind CSS, Radix UI, Recharts.
- Routes in `src/routes.tsx`: Dashboard, Devices, Sensors, Readings, Alerts, Firmware, Settings
- API client in `src/api/api.ts` — all backend calls via axios, base URL hardcoded to `http://localhost:8000`
- Pages in `src/pages/`, reusable components in `src/components/`
- Path alias: `@` → `src/`

**Embedded** (`embeded-src/`) — MicroPython on ESP32. Note the directory is intentionally spelled `embeded-src`.
- `main.py` — WiFi connect, device/sensor registration, reading loop, heartbeat, OTA checks
- `captive_portal.py` — AP mode provisioning (SSID: `PlantSensor_XXXX`, pw: `plantsetup`)
- `config.py` — device-specific config, gitignored, generated by captive portal

## Data Flow

1. ESP32 boots → connects WiFi (or starts captive portal for provisioning)
2. Registers itself via `POST /devices/register_device`, then registers sensors via `POST /sensors/`
3. Main loop: reads moisture → `POST /readings`, heartbeat every 300s, OTA check periodically
4. Backend checks thresholds on new readings → creates alerts if exceeded
5. Frontend polls backend APIs to display devices, readings, alerts

## Key Conventions

- Package manager for frontend is **pnpm** (not npm)
- Backend uses **pip** with `requirements.txt` (no pyproject.toml)
- Database is SQLite — no migrations system; schema defined in `models.py`, tables auto-created
- Device cascade delete: deleting a device removes its sensors, readings, and alerts
- Sensor readings are currently **simulated** (random values) — real ADC wiring not yet implemented
- CORS is fully open (`*`) in backend — development only
